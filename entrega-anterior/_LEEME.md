# Entrega anterior — archivo de referencia

Esta carpeta conserva **la propuesta original** que se le presentó al cliente (alcance amplio, 3 roles: Inversor, Asesor de la desarrolladora y Superuser UMBRAL con flujo de aprobación editorial).

El cliente aprobó la dirección pero, por precio y tiempos, **acortó el alcance**. El desarrollo real arranca con el MVP (ver `/uploads/MVP.md`), que vive en la raíz del proyecto.

> Se mantiene como contexto y memoria de diseño. No es el alcance a construir.

**Diferencias clave con el MVP:**

| | Entrega anterior (acá) | MVP (raíz) |
|---|---|---|
| Roles | Inversor · Asesor desarrolladora · Superuser | Inversor · Admin UMBRAL |
| Backoffice | Desarrolladora carga → UMBRAL aprueba/publica | UMBRAL opera todo, directo |
| App inversor | Portfolio multi-proyecto, lenguaje financiero (TIR, renta) | Un desarrollo, **valorización por m²**, avance de obra, cuotas |
| Alta inversor | — | Código de referencia único + onboarding |
| Flujos nuevos | — | Vender · Nueva duda · FAQ inline |

Archivos: `UMBRAL_MVP_Pantallas.html` (canvas de pantallas), `UMBRAL_Propuesta_MVP*.html` (propuesta), y los `.jsx` de cada rol.
